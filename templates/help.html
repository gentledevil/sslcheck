Comment ça marche ?

  • Lors de la mise en place d'un certificat SSL, l'ajouter à la liste des hosts à checker
  • Les certificats présents dans la base sont régulièrement checkés par un cron
  • Des alertes sont envoyés par mail lorsque des certificats invalides sont détectés
  • Il est possible d'ignorer un certificat lors des checks et dans ce cas remplir la base manuellement

Que signifient les différents tests ?

  • DNS valid / Net OK : Le hostname de la machine est-il résolvable, la machine est-elle joignable sur le port indiqué ?
  • Cert valid : le certificat est-il correctement formé (chaîne de certification, ...)
  • Certname match : le CN du certificat correspond-il au hostname de la machine ?
  • Expire days/Expiration date : quand le certificat va-t-il expirer ?
  • Revoked : le serial du certificat apparait-il dans la liste de révocation (CRL) associée ?
  • Heartbleed : le serveur est-il vulnérable à la faille heartbleed ?

    • Les checks sont dépendants les uns des autres. Si par exemple le certificat n'est pas valide, le check sur la date d'expiration n'aura pas lieu.
